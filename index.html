A proxy is required to crawl most external sites — use automatic to get started immediately.
?

Browsers block requests to other websites for security reasons (CORS). A proxy sits in between — Canopy asks the proxy to fetch the page, and the proxy returns it. This lets the Crawler crawl any site.

Automatic — uses the built-in Canopy proxy. Works immediately, no setup needed.
Manual — use your own Cloudflare Worker proxy if you prefer to host it yourself.

▶ Setting up your own Cloudflare Worker
1. Create a free account at cloudflare.com
2. Go to Workers & Pages → Create → Create Worker
3. Replace the default code with the contents of canopy-worker.js
4. Edit ALLOWED_ORIGIN to match your site's URL
5. Click Deploy — copy the Worker URL into the manual proxy field

What is Canopy?

Canopy is a sitemap toolkit.

It crawls a live website, maps the structure of every page it can reach, and generates a visual tree diagram — useful for understanding a site before a redesign, migration, or audit, and for sharing that structure with a client.

The workflow

🕷️ Crawler — point it at any URL and Canopy walks all reachable internal pages. Use the Workbench to filter out what you don't need. When done, hit → build diagram.

🌳 Diagram — an interactive tree laid out by section and depth. Rectangular nodes show the URL slug. Pan and zoom to explore.

🍂 Export — download as PNG or SVG for presentations, CSV for spreadsheets, or JSON for developers.

A note on crawling

A built-in Cloudflare Worker proxy handles CORS restrictions so Canopy can crawl any public site. Switch to manual proxy in the Crawler settings if you'd prefer to use your own.

Privacy

Canopy runs entirely in your browser. The only exception is the proxy feature: when crawling a site, page requests are routed through a Cloudflare Worker so the browser can reach external sites. Only the URLs being fetched are passed through.
